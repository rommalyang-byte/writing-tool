# 写作辅助工具 — 产品规格 SPEC v0.1

> **定位**：接入大模型的「剧情细化 + 资料填充」助手。AI 产出的是结构化的细纲 / 节拍 / 资料，供作者审核采纳；**不生成正文**。正文由作者自行完成。

---

## 0. 核心原则

- **人在环**：AI 永远只给「建议」，由作者 接受 / 编辑 / 拒绝 后才进入数据；系统不自动改写已采纳内容。
- **本地优先**：数据存本地硬盘；Markdown 导出作为备份与人类可读稿。
- **图形化**：拖曳排序、删除；以卡片 / 树 / 看板呈现。
- **多模型可换**：统一接入层，运行时切换不同厂商的模型。

---

## 1. 数据模型

### 1.1 实体层级

```
作品 Project
├─ 大纲 Outline（卷 / 篇）
│  └─ 细纲 Section（章 / 段落级）
│     └─ 节拍 Beat（最小单元）
├─ 人物 Character
├─ 人物关系 Relationship（边）
├─ 资料条目 Reference（设定 / 技术 / 历史 / 真实事件）
└─ 时间轴 Timeline（= 节拍现实日期的派生视图）
```

### 1.2 人物 Character

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | 短文本 | 主显示字段 |
| 出生年月 | 日期（年-月） | |
| 年龄 | 数字（派生） | 默认由出生年月自动算，可手动覆盖 |
| 性格优点 | 短文本 / 多值 | |
| 性格缺点 | 短文本 / 多值 | |
| 职业 | 短文本 | |
| 工作 / 学习单位 | 短文本 | 原「工作单位」已并入此项 |
| 担任职务 | 短文本 | |
| 家庭住址 | 短文本 | 位置类 |
| 日常活动区域 | 短文本 | 位置类 |
| 日常爱好 | 列表 | 默认 3 项，可增减 |
| 特殊口癖 | 短文本 | |
| 特殊行为习惯 | 短文本 | |
| 人设标签 | 列表 | 默认 3 项，可增减（原「基本性格」） |
| 外貌·明星原型 | 短文本 | |
| 外貌特征 | 列表 | 默认 3 项，可增减 |
| 行为动机 | 中长文本 | |
| 人物小传 | 长文本 | 全文单独存；表内只显示截断预览 + 点击展开 |

**表格 UI**：单元格行高 / 列宽随内容自适应伸缩；一行 = 一个人物，可增删行。

### 1.3 大纲 / 细纲 / 节拍

公共字段：`id`、`父节点 id`、`排序序号`、`标题`、`内容`、`来源标记`（AI建议 / 已采纳 / 手改）。

- **大纲 Outline**：卷 / 篇级。
- **细纲 Section**：章 / 段落级；含 `后续影响 / 伏笔钩子` 字段。
- **节拍 Beat**，额外字段：
  - `现实日期 / 时间范围`（供 F3 检索）
  - `后续影响 / 伏笔钩子` = { 文本, 目标节拍链接(payoff), 兑现状态(人工勾选：已回收 / 悬空) }
  - `注入资料`（F3 检索到的真实事件，列表，引用 Reference）
  - `过时标记`（F2 用）

### 1.4 人物关系 Relationship（边）

以**主角为中心**的星型结构。

| 字段 | 类型 | 说明 |
|------|------|------|
| A | 引用人物 | 默认主角 |
| B | 引用人物 | |
| 关系类型 | 枚举 + 自定义 | 家人 / 同事 / 朋友 / 师生 / 敌对 … |
| 戏剧冲突 | 文本 | 可手填，或选中相关细纲让模型辅助提炼 → 采纳 |
| 变化时间点 | 预留 | MVP 留空 = 静态关系；时间维度后续加 |

### 1.5 资料条目 Reference

字段：`类型`（设定 / 技术 / 历史 / 真实事件）、`标题`、`内容`、`来源`（手填 / AI / F3 检索）、`关联对象`（可挂到节拍 / 人物 / 章节）。

## 2. 功能

### F1 逐级细化 — MVP
选中大纲 → 生成细纲建议；选中细纲 → 生成节拍建议。建议以卡片出现，作者 接受 / 编辑 / 拒绝 后入表。调用当前选定模型。

### F2 受控级联同步 — MVP，核心
- **结构改动**（增删 / 移动 / 重排节点）：子节点链接自动联动，确定性操作。
- **内容改动**（改了某节讲什么）：下游节拍标「过时」。作者改完大章 → 点「确认修改节拍」→ 模型生成**修改样张**（与现节拍的 diff 预览）→ 作者手工确认 → 才替换节拍表。
- **触发**：手动 + 防抖，非实时（避免每次编辑都调模型，又慢又贵）。
- 重生时**保留并重建伏笔链接**，变动一并进 diff。

### F3 真实事件检索注入 — 后续
按节拍的 `现实日期 / 时间范围` 联网检索当时前后的真实事件，作为资料注入节拍以增真实感，作者审核采纳。需要联网检索能力（模型自带 web search，或独立搜索 API）。

### F4 伏笔追踪 — MVP 基础版
- setup ⟶ payoff 链接，在节拍视图直接显示连线 / 标记。
- 状态：已回收 / 悬空。**是否兑现由人工勾选**；系统只判断「有无指向、目标是否还在」。
- 随 F2 改动自动更新：目标节拍被删 → 该伏笔标「悬空」并提醒；移动 / 重排 → 链接跟随不断。

### F5 时间线剧情关系图 — 后续
按剧情时间线生成思维导图式概述，以主角为中心画人物关系（标关系类型 + 戏剧冲突）。读 人物关系表 + 时间轴 渲染。

### 横切：多模型接入层 — MVP 建抽象，先接 1–2 家
- 一个 **OpenAI 兼容客户端** + provider 配置表 `{ base_url, api_key, model }`，覆盖 ChatGPT、DeepSeek、千问（百炼兼容端点）、Gemini（OpenAI 兼容端点）、Moonshot。
- Claude（Anthropic）与 GLM（智谱）走各自 SDK 分支，或用其 OpenAI 兼容路径。
- 可选更省事：接 OpenRouter 这类聚合网关，一个 Key 调全部。
- API Key 由作者在应用内填写，加密存本地。

## 3. UI 视图

- **大纲-细纲-节拍**：可展开树 + 看板，支持拖曳排序 / 删除。
- **人物表**：Excel 式表格，自适应单元格，小传截断 + 展开。
- **节拍视图**：卡片 + 伏笔连线 + 过时标记。
- **AI 建议**：卡片式，接受 / 编辑 / 拒绝。
- **设置**：模型 provider 配置与 Key 管理。
- （后续）人物关系图 / 时间线思维导图。

---

## 4. 存储与导出

- **工作存储**：SQLite（结构化，便于关系查询与级联逻辑）。
- **备份 / 可读稿**：一键导出整个项目为 Markdown 文件到本地。
- **Google Drive 自动同步**：可选后续（把导出的 md 推送到 Drive）。

---

## 5. 技术选型（建议，可调）

- **形态**：本地运行的桌面级应用（浏览器纯前端无法自由读写本地硬盘）。
  - 推荐 **Tauri 2 + React + TypeScript**（原生文件访问、体积小、跨平台）；
  - 或 **Electron**（全 JS，工具链更熟悉则选它）。
- **数据**：SQLite（`tauri-plugin-sql` 或 `better-sqlite3`）。
- **拖拽**：dnd-kit。**树 / 看板**：react-arborist 或自绘。**关系图（后续）**：React Flow 或 Cytoscape。
- **LLM**：统一 `ModelProvider` 抽象 + provider 配置表 + OpenAI 兼容客户端。
- **F3 搜索**：模型自带 web search，或 Tavily / Brave Search API。

---

## 6. 已确认的默认决策

- 删除「性别」字段；保留性格优点 / 缺点；「基本性格」→「人设标签」。
- 年龄由出生年月自动算，可手动覆盖。
- 「三项」字段默认 3、可增减。
- 「工作单位」并入「工作 / 学习单位」。
- 「项目数据」指结构化数据，不含作者在别处写的正文。
- F2 用「标过时 + 样张确认」流程，手动触发。
- 伏笔兑现人工勾选。
- 备份 = Markdown 导出；Google Drive 同步留后续。
- 美国三家 = ChatGPT / Claude / Gemini；国产 = DeepSeek / GLM / 千问 等。

---

## 7. 范围（MVP vs 后续）

**MVP**：作品 / 大纲 / 细纲 / 节拍 CRUD + 人物表 + F1 + F2 + F4（基础）+ 模型接入抽象（先接 1–2 家）+ SQLite + Markdown 导出 + 拖拽 UI。

**后续**：F3 真实事件注入、人物关系表 + F5 关系图 / 时间线、多模型全接入、关系时间维度、Google Drive 同步。

**明确不做**：正文编辑器 / 自动写作、协作 / 多用户、排版导出。
