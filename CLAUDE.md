# CLAUDE.md

本文件是本项目给 Claude Code 的常驻说明。完整需求与数据模型见 @docs/SPEC.md（唯一事实来源）。

## 项目一句话

一个**本地运行**的「剧情细化 + 资料填充」写作助手。AI 只产出结构化建议（细纲 / 节拍 / 资料），由作者审核采纳。**不写正文、不做自动写作。**

## 技术栈

- Tauri 2 + React + TypeScript（桌面级，需本地文件访问）
- SQLite（`tauri-plugin-sql`）存项目数据
- dnd-kit 拖拽；树 + 看板用 react-arborist 或自绘；React Flow 用于后续关系图
- LLM：统一 `ModelProvider` 抽象 + OpenAI 兼容客户端 + provider 配置表

## 不可违反的硬规则

1. **AI 输出永远是「建议」**，经作者 接受 / 编辑 / 拒绝 后才写入数据。禁止任何路径自动改写已采纳内容。
2. **F2 级联**：内容改动只「标记下游节拍为过时」并生成 diff 样张，等作者手动点确认才替换。禁止静默覆盖手改过的节拍。触发为手动 + 防抖，禁止实时调用模型。
3. **F4 伏笔「是否兑现」只由人工勾选**；系统只判断有无指向、目标是否存在，禁止让模型推断兑现。
4. 不实现：正文编辑器、自动写作、协作 / 多用户、排版导出（见 SPEC §7 范围）。
5. **API Key** 由用户在应用内填写、加密存本地。禁止在代码、配置、本文件、提交记录中写入任何真实 key。
6. MVP 阶段不写 F3（真实事件注入）、F5（关系图 / 时间线）；先把抽象与字段留好即可。

## 数据模型要点（细节见 SPEC §1）

- 层级：作品 > 大纲 > 细纲 > 节拍；旁挂 人物、人物关系、资料条目。
- 节拍含：现实日期 / 时间范围、伏笔钩子 `{ 文本 + 目标节拍链接 + 兑现状态(人工) }`、注入资料、过时标记。
- 人物关系 = 以主角为中心的边：关系类型 + 戏剧冲突 + 预留变化时间点（MVP 静态）。

## 代码约定

- TypeScript strict；禁止 `any`，用 `unknown` + 类型守卫。
- 业务逻辑（级联 / diff / 伏笔）集中在 `src/lib`，与 UI 解耦，必须可单测。
- 目录结构见 SPEC §8；新功能放对应 `src/features/` 子目录。
- 代码格式交给 linter / formatter，不在本文件写格式规则。

## 工作方式

- 严格按 @docs/SPEC.md 实现；需求模糊或 SPEC 未覆盖时**先问，不要自行编造需求或字段**。
- 改动核心数据结构或级联逻辑前，先说明方案再动手。
- 任务完成前跑构建与测试（命令见下）。

## 常用命令

- 安装依赖：`npm install`
- 开发运行：`npm run tauri:dev`
- 构建打包：`npm run tauri:build`
- TypeScript 检查：`npx tsc --noEmit`
- Vite 前端构建：`npx vite build`

> 脚手架已初始化（Tauri 2 + React + TypeScript + SQLite）。首次 `tauri:build` 需编译 Rust 后端（~485 个 crate，约 5-15 分钟）。
